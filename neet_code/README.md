# NeetCode 150 -- Java Solutions

Every problem is solved twice:
1. **Brute Force** -- straightforward, usually O(N^2) or worse
2. **Optimal** -- best known time/space complexity

All code is idiomatic Java 21. Compile: `javac File.java` (Java 17+ recommended).

## Java Notes

These solutions favor idiomatic Java 21 over a literal C++ transcription:

- `vector<int>` becomes `int[]` (fixed size) or `List<Integer>` / `ArrayList<>`.
- `unordered_map` becomes `HashMap`; `map` (ordered) becomes `TreeMap`.
- `unordered_set` becomes `HashSet`; `set` becomes `HashSet` (or `TreeSet` when ordering matters).
- `priority_queue` becomes `PriorityQueue` (min-heap by default; use `Collections.reverseOrder()` or a comparator for a max-heap).
- `pair`/`tuple` becomes `int[]` or a small `record`.
- Recursive `std::function` lambdas become private helper methods; mutable accumulators captured by reference are passed explicitly or held as instance fields.
- C++ unsigned bit tricks use Java's logical shift `>>>` to mirror unsigned semantics.

## Complexity Summary by Category

| Category | Typical Brute | Typical Optimal |
|----------|--------------|----------------|
| Arrays & Hashing | O(N^2) | O(N) with hash map |
| Two Pointers | O(N^2) | O(N) |
| Sliding Window | O(N^2) | O(N) |
| Stack | O(N^2) | O(N) monotonic stack |
| Binary Search | O(N) | O(log N) |
| Linked List | varies | O(N) one-pass |
| Trees | O(N^2) | O(N) DFS/BFS |
| Heap | O(N log N) sort | O(N log K) heap |
| Backtracking | exponential | exponential (pruned) |
| Graphs | O(V*E) | O(V+E) BFS/DFS |
| 1-D DP | O(2^N) recursion | O(N) tabulation |
| 2-D DP | O(2^(M+N)) | O(M*N) tabulation |
| Greedy | O(N^2) | O(N log N) |
| Intervals | O(N^2) | O(N log N) sort |
